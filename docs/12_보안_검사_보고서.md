# ClawDroid 보안 검사 보고서

> 작성일: 2026-04-20
> 범위: `app/` 모듈 전체 (소스 + 매니페스트 + 네트워크 설정 + 의존성)
> 기준: OWASP Mobile Top 10 (2024), Android Security Best Practices, MASVS L1

---

## 0. Executive Summary

| 항목 | 결과 |
|---|---|
| 전체 보안 등급 | **B+ (양호)** |
| Critical 이슈 | **0** |
| High 이슈 | **2** |
| Medium 이슈 | **5** |
| Low / Info 이슈 | **6** |
| 즉시 수정 필요 | SEC-H1, SEC-H2 |

ClawDroid는 **EncryptedSharedPreferences · BiometricPrompt · Network Security Config · `allowBackup=false` · R8 난독화** 등 안드로이드 보안 기준선을 충실히 따르고 있다. 다만 **DB 미암호화**, **로그 민감정보 노출 가능성**, **SYSTEM_ALERT_WINDOW 사용**, **alpha 보안 라이브러리 의존** 등 중위험 항목을 보완해야 프로덕션 배포 기준(MASVS L2)을 만족할 수 있다.

## 3. 발견 이슈 상세

### 🔴 High

#### SEC-H1 — Room 데이터베이스 미암호화
- **위치**: `core/data/db/ClawDroidDatabase.java`
- **내용**: `clawdroid.db`에 대화/메시지/ToolCall 전체가 평문 저장. 디바이스 루팅 또는 `adb backup`이 아닌 기기 탈취 시 노출 가능.
- **영향**: 사용자 프라이버시, 기업 데이터 포함 대화 유출.
- **CVSS (추정)**: 6.5 (Medium-High)
- **권장 조치**:
  1. `net.zetetic:android-database-sqlcipher` + Room의 `SupportFactory` 적용.
  2. 암호화 키는 Android Keystore로 보호되는 별도 키(EncryptedSharedPreferences 내부 또는 `KeyGenParameterSpec`).
  3. 기존 DB → 암호화 DB 1회 마이그레이션 경로 제공.

#### SEC-H2 — HttpLoggingInterceptor BODY 레벨로 API 키/토큰 로그 노출 가능
- **위치**: `app/di/NetworkModule.java`
- **내용**: OkHttp 로깅 인터셉터가 `BODY` 레벨일 경우 `Authorization`, `x-api-key`, `Bot {token}` 등이 Logcat에 기록됨. debug 빌드라도 공개 기기·분석 툴에서 유출 위험.
- **영향**: API 키 탈취 → 과금·데이터 접근.
- **권장 조치**:
  1. 릴리스 빌드: `HttpLoggingInterceptor.Level.NONE` (BuildConfig.DEBUG 가드 확인).
  2. debug 빌드도 `redactHeader("Authorization")`, `redactHeader("x-api-key")` 적용.
  3. 요청 바디에 토큰이 포함되는 Slack/Discord 케이스는 커스텀 인터셉터로 마스킹.

---

### 🟠 Medium

#### SEC-M1 — alpha/beta 보안 라이브러리 의존
- **위치**: `gradle/libs.versions.toml`
- **현재**: `androidx.security:security-crypto:1.1.0-alpha06`, `biometric:1.2.0-alpha05`
- **권장**: 프로덕션 릴리스 전에 stable 릴리스 확인 및 업그레이드. 최소한 의존성 버전 고정 + changelog 모니터링.

#### SEC-M2 — SYSTEM_ALERT_WINDOW 권한
- **위치**: `AndroidManifest.xml`, `FloatingOverlayService`
- **위험**: 오버레이 공격(탭재킹, 클릭재킹)에 악용 가능한 고위험 권한. Google Play 정책상 사유 소명 필요.
- **권장**:
  1. Play Console에 사용 목적 선언.
  2. 런타임에 `Settings.canDrawOverlays()` 확인 후 요청.
  3. 민감 입력(PIN, API 키) 화면에서는 `FLAG_SECURE` 설정.

#### SEC-M3 — 사용자 입력 → Tool 실행 시 인자 검증
- **위치**: `feature/tools/tool/ToolExecutor.java`, `builtin/BrowserTool.java` 등
- **위험**: AI가 생성한 function call 인자(URL, 파일 경로, intent action)를 그대로 실행 → Intent Redirection, Path Traversal, SSRF 위험.
- **권장**:
  1. `BrowserTool`: `Uri` 파싱 후 scheme allowlist(`https`, `http`), 호스트 차단 리스트.
  2. `FileManagerTool`: `File.getCanonicalPath()`로 경로 정규화 후 앱 전용 디렉터리 범위 검증.
  3. `AppLauncherTool`: 패키지명 allowlist + `resolveActivity` 검사.
  4. 모든 Tool: 인자 길이·타입 제한.

#### SEC-M4 — PIN 브루트포스 보호 부재 여부
- **위치**: `feature/settings/security/PinManager.java`, `LockActivity`
- **위험**: 시도 횟수 제한·지수 백오프 없으면 4~6자리 PIN 브루트포스 가능.
- **권장**:
  1. 실패 5회 시 1분 잠금, 10회 시 5분 등 지수 증가.
  2. 누적 실패 30회 시 데이터 와이프 옵션(사용자 설정).
  3. 상태는 EncryptedSharedPreferences에 저장.

#### SEC-M5 — 대화/음성 로그 보존 정책 명시 부족
- **위치**: `app/worker/AutoDeleteWorker.java`, 음성 임시파일
- **현재**: 자동 삭제 Worker는 존재하나 기본 주기/보존 기간 UX 노출 여부 불명확.
- **권장**:
  1. 설정 화면에 "보존 기간" 명시 (7/30/90일/무기한) + 기본값 30일.
  2. STT 중간 오디오 캐시는 세션 종료 시 즉시 삭제(`cacheDir`).
  3. 개인정보처리방침에 명기.

---

### 🟡 Low / Info

#### SEC-L1 — `android:allowBackup="false"` 확인 ✅ (이미 적용)
- 추가: `android:dataExtractionRules` (API 31+) 명시 권장.

#### SEC-L2 — ProGuard rules에 Hilt 생성 클래스 명시 부재
- **권장**: `-keep class dagger.hilt.** { *; }`, `-keep @dagger.hilt.android.AndroidEntryPoint class *` 등 명시적 keep 규칙 추가(최신 Hilt는 AGP 통합으로 자동이지만 방어적 명시 권장).

#### SEC-L3 — Network Security Config cleartext 예외(localhost)
- localhost/127.0.0.1 한정이므로 수용 가능. 단, **릴리스 빌드 전용**으로 분리(`debug/res/xml/network_security_config.xml`) 고려.

#### SEC-L4 — Deep Link / Intent Filter 감사
- 현재 `MainActivity`에만 LAUNCHER intent-filter. 추후 OAuth 콜백용 intent-filter 추가 시 반드시 `android:autoVerify="true"` + `App Links` 도메인 검증.

#### SEC-L5 — TileService `BIND_QUICK_SETTINGS_TILE`
- `VoiceChatTileService exported="true"` + 시스템 전용 permission 조합 ✅ (올바름).

#### SEC-L6 — FileProvider authorities
- `${applicationId}.fileprovider` + `exported="false"` + `grantUriPermissions="true"` ✅ (올바름). `file_paths.xml`에서 필요한 경로만 제한되어 있는지 정기 감사 권장.

## 4. 데이터 흐름별 보안 상태

```
[사용자 PIN]
   ↓ SHA-256 + 16B SecureRandom Salt
   ↓ Base64
   → SharedPreferences (non-encrypted)  ⚠️ 권장: EncryptedSharedPreferences로 이동

[API Keys (OpenAI/Gemini/Custom)]
   ↓ 사용자 입력
   → EncryptedSharedPreferences (AES-256-GCM)  ✅

[채널 토큰 (Telegram/Discord/Slack)]
   ↓ 입력
   → EncryptedSharedPreferences 또는 ChannelEntity  ⚠️ 저장 경로 확인 필요

[대화 메시지]
   ↓ Room DAO Insert
   → clawdroid.db (평문)  🔴 SEC-H1

[네트워크 요청 (Retrofit/OkHttp)]
   ↓ HTTPS 강제 (localhost 예외)
   ↓ HttpLoggingInterceptor BODY?  🔴 SEC-H2
   → 외부 API

[음성 오디오]
   ↓ SpeechRecognizer 세션
   → (캐시) 세션 종료 삭제 여부 확인 필요  ⚠️ SEC-M5
```

## 6. 지속적 보안 운영 권고

1. **CI 보안 파이프라인**
   - Gradle `dependencyCheck` (OWASP Dependency-Check) 또는 GitHub Dependabot 활성화.
   - `detekt` + `lint` 보안 규칙 적용 (`hardcoded-credential`, `logging-sensitive`).
   - `MobSF` 주간 자동 스캔.
2. **비밀정보 관리**
   - `local.properties`·keystore 파일 `.gitignore` 유지 확인.
   - Git pre-commit hook으로 토큰·키 커밋 차단(gitleaks).
3. **릴리스 서명**
   - v2/v3 서명 체계, Play App Signing 사용.
   - `keystore.properties.template`만 커밋, 실제 `keystore.properties`는 제외.
4. **취약점 공개 채널**
   - `SECURITY.md`에 보안 이슈 리포트 경로 명시.
5. **사용자 대면 투명성**
   - 설정 화면에 "어떤 데이터가 어디에 저장되는지" 요약 + 전체 삭제 버튼.

## 8. 결론

ClawDroid는 안드로이드 플랫폼 보안 기본기를 잘 갖춘 앱이다. **SEC-H1(DB 암호화)** 과 **SEC-H2(로그 민감정보 차단)** 두 항목만 해소하면 개인 정보보호 관점의 실질적인 방어선이 완성되며, **SEC-M3(Tool 인자 검증)** 은 AI 기반 앱의 고유 공격면이므로 다른 보안 이슈보다 우선적으로 다룰 것을 권고한다.
