### 음성 대화 예시

```
1. 사용자: "Hey Claw" (Wake Word)
2. ClawDroid: [딩] (리스닝 시작 효과음)
3. 사용자: "내일 서울 날씨 어때?"
4. STT → "내일 서울 날씨 어때?"
5. AI 처리 → "내일 서울은 맑고 최고 기온 22도 예상입니다."
6. TTS → 음성 출력
```

## 🔗 F04. 멀티채널 연동 (Phase 3)

| ID | 기능 | 우선순위 | 설명 |
|---|---|---|---|

| F04-2 | Discord 봇 연동 | P1 | Bot Token으로 Discord Gateway WebSocket 연결 |
| F04-3 | Slack 봇 연동 | P1 | Slack App으로 Events API + Web API 연결 |
| F04-4 | 채널 관리 UI | P0 | 채널 추가/삭제/상태 확인 화면 |
| F04-5 | 상시 연결 | P0 | Foreground Service로 채널 연결 유지 |
| F04-6 | DM 페어링 | P1 | 미인증 사용자 → 페어링 코드 발급 → 승인 |
| F04-7 | 채널별 페르소나 | P2 | 채널마다 다른 시스템 프롬프트 설정 가능 |
| F04-8 | Gateway 모드 | P2 | 별도 Gateway 서버에 노드로 연결 |

### 독립 모드 vs Gateway 모드

| 항목 | 독립 모드 (Standalone) | Gateway 모드 |
|---|---|---|
| **서버** | 불필요 | Gateway 서버 필요 (Node.js/Ktor) |
| **채널 연결** | 앱이 직접 Bot API 연결 | 서버가 채널 연결 관리 |
| **상시 가동** | 기기 켜져 있을 때만 | 서버가 24/7 유지 |
| **배터리** | Foreground Service 소모 | 앱은 경량 WebSocket만 |
| **적합 사례** | 개인 사용, 소수 채널 | 24/7 봇 운영, 다수 채널 |

### Function Calling 흐름

```
사용자: "내일 오후 2시에 치과 예약 알림 만들어줘"
   ↓
AI 판단: alarm 도구 호출 필요
   ↓
Function Call: { "tool": "alarm", "params": { "title": "치과 예약", "datetime": "2026-04-18T14:00:00" } }
   ↓
도구 실행: AlarmManager로 알람 등록
   ↓
AI 최종 응답: "내일 오후 2시에 '치과 예약' 알림을 설정했습니다."
```

### SKILL.md 포맷 (OpenClaw 호환)

```markdown
---
name: weather
description: 날씨 정보를 조회합니다
tools:
  - browser
triggers:
  - 날씨
  - weather
  - 기온
---

# 날씨 스킬

사용자가 날씨를 물으면 browser 도구로 날씨 정보를 검색하여 답변합니다.

## 시스템 프롬프트 추가

날씨 관련 질문에는 항상 지역, 현재 기온, 체감 온도, 강수 확률을 포함하세요.
```
