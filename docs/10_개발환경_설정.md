## 📱 Gemini Nano 테스트 요구사항

> [!warning] Gemini Nano는 에뮬레이터에서 동작하지 않음
> 반드시 **실제 기기**에서 테스트해야 하며, AICore 시스템 서비스가 필요합니다.

### 지원 기기

| 기기 | Android 버전 | AICore 상태 |
|---|---|---|
| Pixel 8 Pro | Android 14+ | ✅ 기본 포함 |
| Pixel 9 / 9 Pro | Android 15+ | ✅ 기본 포함 |
| Samsung Galaxy S24+ | Android 14+ (One UI 6.1+) | ✅ 기본 포함 |
| Samsung Galaxy S25 | Android 15+ | ✅ 기본 포함 |

### AICore 설정

```bash
# 1. AICore가 설치되어 있는지 확인
adb shell pm list packages | grep aicore

# 2. AICore 앱을 최신 버전으로 업데이트 (Play Store에서)

# 3. Gemini Nano 모델 다운로드 트리거
adb shell am broadcast \
  -a com.google.android.aicore.ACTION_DOWNLOAD_MODEL \
  -n com.google.android.aicore/.ModelDownloadReceiver

# 4. 모델 다운로드 상태 확인
adb logcat -s AICore
```

### AndroidManifest.xml 설정

```xml
<application ...>
    <!-- Gemini Nano 사용 선언 -->
    <meta-data
        android:name="com.google.android.aicore.SUPPORTED"
        android:value="true" />

    <!-- ML Kit GenAI 다운로드 설정 -->
    <meta-data
        android:name="com.google.mlkit.genai.DEPENDENCIES"
        android:value="prompt_api,summarization,image_description,speech_recognition" />
</application>
```

## 🔑 API 키 설정

### local.properties (Git에 커밋하지 않음)

```properties
# Gemini Cloud API Key (Google AI Studio에서 발급)
GEMINI_API_KEY=your_gemini_api_key_here

# OpenAI API Key (선택)
OPENAI_API_KEY=your_openai_api_key_here

# Telegram Bot Token (선택)
TELEGRAM_BOT_TOKEN=your_bot_token_here

# Discord Bot Token (선택)
DISCORD_BOT_TOKEN=your_bot_token_here
```

### BuildConfig로 주입

```groovy
// app/build.gradle
android {
    buildFeatures {
        buildConfig true
    }

    defaultConfig {
        // local.properties에서 읽기
        def properties = new Properties()
        def localFile = rootProject.file("local.properties")
        if (localFile.exists()) {
            properties.load(localFile.newDataInputStream())
        }

        buildConfigField "String", "GEMINI_API_KEY",
            "\"${properties.getProperty('GEMINI_API_KEY', '')}\""
    }
}
```

> [!warning] 프로덕션에서는 BuildConfig 대신 EncryptedSharedPreferences 사용
> BuildConfig는 APK에 평문 포함되므로 개발/디버그용으로만 사용.
> 프로덕션은 사용자가 앱 내 설정에서 직접 입력 → EncryptedSharedPreferences에 저장.

## 🏗️ 빌드 & 실행

### 빌드 명령어

```bash
# 디버그 빌드
./gradlew assembleDebug

# 릴리스 빌드
./gradlew assembleRelease

# 빌드 + 설치 (연결된 기기)
./gradlew installDebug

# 전체 테스트
./gradlew test

# 린트 체크
./gradlew lint

# 클린 빌드
./gradlew clean assembleDebug
```

### 실행 구성 (Run Configuration)

| 구성 | 용도 |
|---|---|
| `app` | 메인 앱 실행 (실제 기기 권장) |
| `app (no Nano)` | Gemini Nano 없이 실행 (에뮬레이터 가능) |
| `All Tests` | 전체 단위 테스트 |
| `UI Tests` | Espresso UI 테스트 (기기 필요) |

## 🔏 서명 설정

### keystore 생성

```bash
keytool -genkey -v \
  -keystore clawdroid-release.keystore \
  -alias clawdroid \
  -keyalg RSA -keysize 2048 \
  -validity 10000
```

### 서명 설정 (app/build.gradle)

```groovy
android {
    signingConfigs {
        release {
            def props = new Properties()
            def propsFile = rootProject.file("keystore.properties")
            if (propsFile.exists()) {
                props.load(propsFile.newDataInputStream())
            }
            storeFile file(props.getProperty('storeFile', ''))
            storePassword props.getProperty('storePassword', '')
            keyAlias props.getProperty('keyAlias', '')
            keyPassword props.getProperty('keyPassword', '')
        }
    }

    buildTypes {
        release {
            minifyEnabled true
            shrinkResources true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'),
                'proguard-rules.pro'
            signingConfig signingConfigs.release
        }
    }
}
```

## 🚀 CI/CD — GitHub Actions

### .github/workflows/android.yml

```yaml
name: Android CI

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up JDK 17
        uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'

      - name: Setup Gradle
        uses: gradle/actions/setup-gradle@v4

      - name: Run Tests
        run: ./gradlew test

      - name: Run Lint
        run: ./gradlew lint

      - name: Build Debug APK
        run: ./gradlew assembleDebug

      - name: Upload APK
        uses: actions/upload-artifact@v4
        with:
          name: debug-apk
          path: app/build/outputs/apk/debug/*.apk

  release:
    if: github.ref == 'refs/heads/main'
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up JDK 17
        uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'

      - name: Decode Keystore
        run: echo "${{ secrets.KEYSTORE_BASE64 }}" | base64 -d > clawdroid-release.keystore

      - name: Build Release Bundle
        env:
          STORE_PASSWORD: ${{ secrets.STORE_PASSWORD }}
          KEY_ALIAS: ${{ secrets.KEY_ALIAS }}
          KEY_PASSWORD: ${{ secrets.KEY_PASSWORD }}
        run: ./gradlew bundleRelease

      - name: Upload to Play Store
        uses: r0adkll/upload-google-play@v1
        with:
          serviceAccountJsonPlainText: ${{ secrets.PLAY_SERVICE_ACCOUNT }}
          packageName: com.clawdroid.app
          releaseFiles: app/build/outputs/bundle/release/*.aab
          track: internal
```
