# 01. 프로젝트 개요

> **최종 업데이트**: 2026-04
> [!info] 프로젝트 목표
> OpenClaw에서 영감을 받은 **Android 네이티브 개인 AI 어시스턴트** 앱.
> Gemini Nano 온디바이스 AI를 기본으로 사용하면서, 멀티 모델(Gemini Cloud, OpenAI, Ollama 등)을 선택·폴백 지원.
> 텍스트/음성 대화, 멀티채널 메신저 연동(Telegram/Discord/Slack), 도구·스킬 시스템까지 지원하는 올인원 AI 어시스턴트.

---

## 🎯 핵심 가치 제안

- **Android 네이티브 퍼스트**: Java + XML Views (ViewBinding), 독립 실행 가능 (서버 불필요)
- **온디바이스 AI 기본**: Gemini Nano(AICore)로 인터넷 없이도 AI 대화 가능
- **멀티 모델 지원**: Gemini Nano / Gemini Cloud / OpenAI / Ollama / 커스텀 엔드포인트 자유 선택
- **멀티 채널 연동**: 하나의 AI가 Telegram, Discord, Slack 등에서 동시 응답
- **음성 대화**: 웨이크 워드, STT, TTS로 핸즈프리 AI 어시스턴트
- **도구·스킬 시스템**: 브라우저, 캘린더, 연락처, 카메라 등 기기 기능을 AI가 직접 활용
- **프라이버시**: 온디바이스 처리 우선, 사용자 데이터 외부 전송 최소화
- **독립 + Gateway 모드**: 기기 단독 동작 또는 서버 연결 선택

---

## 🆚 OpenClaw과의 관계

### OpenClaw이란?

[OpenClaw](https://github.com/openclaw/openclaw)은 Node.js 기반 오픈소스 개인 AI 어시스턴트.
Gateway 서버를 중심으로 WhatsApp, Telegram, Discord 등 20+ 채널에 연결하고, 음성 대화(Voice Wake/Talk Mode), 도구/스킬 시스템, 멀티 에이전트 라우팅을 지원.

### ClawDroid의 차별점

| 항목 | OpenClaw | ClawDroid |
|---|---|---|
| **기반 플랫폼** | Node.js Gateway 서버 | **Android 네이티브 앱 (Java)** |
| **AI 모델** | 클라우드 모델 중심 (다중 지원) | **Gemini Nano 온디바이스 기본** + 클라우드 폴백 |
| **서버 의존성** | Gateway 서버 필수 | **독립 모드** (서버 불필요) + Gateway 모드 선택 |
| **오프라인** | 불가 | **완전 오프라인 동작** (Gemini Nano) |
| **프라이버시** | 서버 거쳐 처리 | **기기 내 처리 우선** |
| **설치 편의성** | CLI 설정 필요 (npm 등) | **Play Store에서 설치** |
| **채널 지원** | 20+ 채널 | 주요 채널 우선 (확장 가능) |
| **도구 실행** | 서버에서 실행 | **기기에서 직접 실행** (카메라, GPS 등 하드웨어 접근) |

## 📅 프로젝트 마일스톤

| Phase | 내용 | 기간 | 상태 |
|---|---|---|---|
| Phase 1 | **MVP: AI 채팅** — 프로젝트 스캐폴딩, AI 프로바이더 추상화, Gemini Nano/클라우드 통합, 채팅 UI | 3~4주 | 📋 예정 |

| Phase 3 | **멀티채널** — 채널 프레임워크, Telegram/Discord/Slack 연동, Gateway 모드 | 3주 | 📋 예정 |
| Phase 4 | **도구·스킬** — 도구 프레임워크, Function Calling, 내장 도구, 스킬 시스템 | 2~3주 | 📋 예정 |
| Phase 5 | **고급 기능** — 멀티 세션, 보안, 위젯, CI/CD, Play Store 배포 | 2주 | 📋 예정 |

## 👤 대상 사용자

| 페르소나 | 설명 |
|---|---|
| **개인 AI 사용자** | ChatGPT/Gemini를 자주 쓰지만 프라이버시·오프라인·커스터마이징을 원하는 사용자 |
| **파워 유저** | 멀티채널(Telegram/Discord)에서 AI 봇을 운영하고 싶은 사용자 |
| **개발자** | 커스텀 도구·스킬을 만들어 AI 어시스턴트를 확장하고 싶은 사용자 |
| **프라이버시 중시 사용자** | 대화 데이터를 외부 서버에 보내고 싶지 않은 사용자 |

## 📐 핵심 원칙

1. **Android 네이티브 퍼스트** — 독립 실행이 기본, 서버는 선택
2. **온디바이스 AI 기본** — Gemini Nano → 클라우드 폴백 순서
3. **멀티 모델 개방** — 특정 AI 벤더에 종속되지 않음
4. **프라이버시** — 기기 내 처리 우선, 데이터 외부 전송 최소화
5. **확장 가능** — 도구·스킬·채널을 플러그인처럼 추가
6. **OpenClaw 호환** — 스킬(SKILL.md) 포맷, Gateway 프로토콜 호환 추구
