# ClawDroid 디자인 개선방안

> 작성일: 2026-04-20
> 대상: `app/` 모듈 전체 UI/UX · 시각 디자인 · 디자인 시스템 · 접근성
> 범위: `app/src/main/res/`, `feature/*/ui/`, `design/*/code.html` 시안 대비 구현 갭

---

## 1. 현황 요약

| 항목 | 현재 상태 | 비고 |
|---|---|---|
| UI 프레임워크 | XML Layout + ViewBinding | Jetpack Compose 미도입 |
| 디자인 시스템 | Material 3 (1.12.0) | 커스텀 토큰 부재 |
| 테마 | Single theme (`Theme.ClawDroid`) | Dynamic Color(Material You) 미적용 |
| 타이포그래피 | 기본 Material textAppearance | 전용 타입 스케일 미정의 |
| 다크모드 | 시스템 따름 (추정) | `values-night/` 리소스 부분 구현 |
| 다국어 | 한국어/영어 부분 지원 | `strings.xml` 단일 locale 중심 |
| 접근성 | 최소 수준 | `contentDescription`, TalkBack 미검증 |
| 디자인 시안(HTML) | 14개 화면 | 구현 일치도 70~80% 추정 |

## 2. 주요 문제점 (Pain Point)

### 2.1 디자인 시스템 부재
- 색상·간격·모서리·고도(elevation) 값이 레이아웃 XML에 하드코딩.
- 동일 컴포넌트(예: 메시지 버블, 설정 항목)가 화면마다 재정의되어 일관성 부족.
- `design/*/code.html` 시안의 토큰(예: `--color-primary`, spacing 4/8/12/16)과 리소스 값 간 매핑이 존재하지 않음.

### 2.2 화면별 불일치
- `ChatFragment` ↔ `design/chat_screen_en/code.html`: 스트리밍 인디케이터, 코드블럭 복사 버튼, 롱프레스 컨텍스트 메뉴 위치 등이 시안과 다름.
- `ConversationListFragment`: 검색바/필터칩 상단 고정 스크롤이 시안 대비 단순 리스트.
- `VoiceChatFragment`: 음파/애니메이션 상태(대기·청취·응답)가 시안처럼 3-state 모션 없이 정적 아이콘.
- `ChannelListFragment` ↔ `design/channel_list_en/code.html`: 채널 상태 뱃지(ONLINE/OFFLINE/ERROR)의 시각 대비 약함.

### 2.3 반응형·폼팩터
- 대화면(폴더블 펼침, 태블릿)에서 Fragment 1-pane 유지 → List/Detail 동시 표시(2-pane) 부재.
- `sw600dp`, `w840dp` 리소스 버킷 미정의.
- 가로 방향(landscape)에서 VoiceChat/Chat 입력창 가림 케이스 가능.

### 2.4 상태 처리 UX
- 로딩/빈 상태/에러 상태의 공통 컴포넌트(EmptyView, ErrorView, ShimmerLoader) 없음.
- 네트워크 오류 시 Snackbar 일관성 부족 — Toast, AlertDialog, Snackbar 혼용 추정.

### 2.5 접근성·국제화
- `contentDescription` 누락 아이콘 다수 (RecyclerView item의 아이콘 버튼).
- 터치 타겟 48dp 미만 아이콘 존재 가능성.
- 색 대비(WCAG AA) 비검증 — 특히 secondary/tertiary 텍스트.
- 한국어 외 영어 `values-en/` 분리되어 있지 않을 가능성.

### 2.6 다크모드 & Dynamic Color
- Material You (Android 12+, minSdk 26이므로 조건부 지원) 미적용.
- 다크 팔레트가 단순 반전 수준 — OLED true-black 프리셋 없음.

### 2.7 모션·마이크로 인터랙션
- RecyclerView item 진입/퇴장 애니메이션 부재.
- Fragment 전환 시 Shared Element Transition 미사용.
- 스트리밍 응답 시 타이핑 커서/하이라이트 모션 부재.

## 3. 개선 방안

### 3.1 단기 (1~2주, XML 유지)

| ID | 항목 | 설명 | 산출물 |
|---|---|---|---|
| D-S1 | **디자인 토큰 정의** | `res/values/tokens.xml`에 색상·간격·radius·elevation 토큰화 | `tokens.xml`, `spacing.xml` |
| D-S2 | **Material 3 테마 정비** | `Theme.ClawDroid` → `Theme.Material3.DayNight.NoActionBar` 기반, 5개 Semantic Color Role(`colorPrimary`, `colorSecondaryContainer`, `colorSurfaceVariant` 등) 재정의 | `themes.xml`, `themes.xml (night)` |
| D-S3 | **공통 상태 컴포넌트** | `EmptyView`, `ErrorView`, `LoadingShimmer` 커스텀 뷰 + `<include>` 패턴 | `core/ui/state/` |
| D-S4 | **Snackbar 표준화** | 모든 에러·성공 피드백을 Snackbar로 통일하는 `FeedbackUtil` | `core/ui/FeedbackUtil.java` |
| D-S5 | **아이콘/터치타겟 감사** | 48dp 미만, `contentDescription` 누락 일괄 수정 | Lint baseline |
| D-S6 | **RecyclerView 애니메이션** | `DefaultItemAnimator` → `SlideInUpAnimator` 혹은 `ItemAnimator` 커스텀, DiffUtil 적용 확인 | Adapter 수정 |

### 3.2 중기 (3~6주)

| ID | 항목 | 설명 |
|---|---|---|
| D-M1 | **반응형 레이아웃** | `layout-sw600dp/`, `layout-w840dp/` 버킷 추가, `SlidingPaneLayout`으로 Conversation List+Chat 2-pane 구성 |
| D-M2 | **Dynamic Color** | `DynamicColors.applyToActivitiesIfAvailable()` (Android 12+), 사용자 설정에서 on/off 토글 |
| D-M3 | **다크/OLED 테마** | `values-night/colors.xml`에 true-black 팔레트, 설정 화면에서 Light/Dark/System/OLED 선택 |
| D-M4 | **Motion System** | Material Motion (`MaterialFadeThrough`, `MaterialSharedAxis`)로 Fragment 전환 표준화 |
| D-M5 | **Voice 상태 모션** | Lottie 혹은 `MotionLayout`로 대기→청취→응답 3-state 애니메이션 |
| D-M6 | **접근성 WCAG AA** | Accessibility Scanner 실행 → 색 대비·라벨·터치 타겟 이슈 해결, TalkBack 수동 검증 |
| D-M7 | **국제화 분리** | `values-en/strings.xml`, `values-ko/strings.xml`로 분리, 날짜·숫자·PluralRules 적용 |

### 3.3 장기 (6주 이상)

| ID | 항목 | 설명 |
|---|---|---|
| D-L1 | **Compose 부분 도입** | `Settings`, `PersonaSettings`, `ChannelDetail` 등 폼 중심 화면부터 Compose로 마이그레이션. `ComposeView` 하이브리드 |
| D-L2 | **디자인 시스템 모듈화** | `core:ui-design` 모듈로 분리, 토큰·컴포넌트를 Compose & XML 양쪽에서 공유 |
| D-L3 | **태블릿/폴더블 전용 UI** | WindowSizeClass 기반 `Compact/Medium/Expanded` 레이아웃 분기 |
| D-L4 | **테마 커스터마이징** | 사용자 정의 accent 색상(Persona별 테마 연동) |
| D-L5 | **시안 vs 구현 픽셀 비교** | Figma/HTML 시안 vs 스크린샷 비교 회귀 테스트(Shot/Paparazzi) |

## 4. 화면별 구체 개선 제안

### 4.1 ChatFragment
- 메시지 버블: 발신자 아바타 · 시각 · 상태(전송중/실패) 3요소 분리.
- 코드블럭: 언어 라벨 + 복사 버튼(우상단) + 가로 스크롤.
- 스트리밍 중: 말풍선 하단에 blinking cursor + 취소(정지) FAB.
- 롱프레스: BottomSheet(복사/재생성/공유/삭제).
- 하단 입력창: `TextInputLayout` + 파일 첨부 · 마이크 · 보내기 3버튼 고정.

### 4.2 ConversationListFragment
- 상단 `SearchBar`(Material 3) + 필터칩(All/Pinned/Archived).
- 스와이프 제스처: 좌→보관, 우→삭제(Undo Snackbar).
- 빈 상태 일러스트 + CTA("새 대화 시작").

### 4.3 VoiceChatFragment
- 중앙 원형 음파 애니메이션(amplitude 연동).
- 상태 라벨: "듣는 중..." / "생각 중..." / "답변 중...".
- 하단 실시간 자막(부분 인식 결과 반투명).
- 오버레이 모드 진입 버튼.

### 4.4 ChannelListFragment / ChannelDetailFragment
- 리스트: 플랫폼 로고 · 채널명 · 상태 뱃지(색+아이콘+텍스트).
- 상세: 연결 로그 타임라인, Webhook URL 복사, 재연결 버튼.
- 토큰 입력: `passwordToggleEnabled` + 마스킹.

### 4.5 Settings 계열
- `PreferenceFragmentCompat` → Material 3 Preference Theme.
- 그룹 헤더 대문자 지양, Sentence case + 아이콘.
- 파괴적 액션(로그아웃/데이터 삭제)은 빨간색 + 확인 다이얼로그.

### 4.6 SecuritySettingsFragment
- PIN 설정 플로우: 단계 인디케이터(입력→재확인→완료).
- 생체 인증 토글 옆 상태 배지(등록됨/미등록).
- "위험 권한" 섹션 별도 분리 + 설명 링크.

## 5. 디자인 토큰 샘플 (res/values/tokens.xml 초안)

```xml
<resources>
    <!-- Spacing -->
    <dimen name="space_xs">4dp</dimen>
    <dimen name="space_sm">8dp</dimen>
    <dimen name="space_md">12dp</dimen>
    <dimen name="space_lg">16dp</dimen>
    <dimen name="space_xl">24dp</dimen>
    <dimen name="space_xxl">32dp</dimen>

    <!-- Corner Radius -->
    <dimen name="radius_sm">8dp</dimen>
    <dimen name="radius_md">12dp</dimen>
    <dimen name="radius_lg">20dp</dimen>
    <dimen name="radius_pill">999dp</dimen>

    <!-- Elevation -->
    <dimen name="elev_1">1dp</dimen>
    <dimen name="elev_2">3dp</dimen>
    <dimen name="elev_3">6dp</dimen>

    <!-- Touch target -->
    <dimen name="touch_min">48dp</dimen>
</resources>
```
