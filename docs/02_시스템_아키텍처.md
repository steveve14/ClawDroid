## 🏛️ 아키텍처 선택: 안드로이드 네이티브 (로컬 퍼스트)

> [!info] 왜 Android 네이티브 + 온디바이스 AI인가?
>
> - **완전한 오프라인 동작**: Gemini Nano로 네트워크 없이 AI 대화 가능
> - **프라이버시**: 대화 데이터가 기기 밖으로 나가지 않음 (온디바이스 모드)
> - **하드웨어 접근**: 카메라, GPS, 연락처, 캘린더 등 Android API 직접 활용
> - **음성 통합**: ML Kit STT, Android TTS, Wake Word 네이티브 지원
> - **Foreground Service**: 채널 연결 상시 유지 가능
> - **Play Store 배포**: 사용자가 쉽게 설치·업데이트

## 🔄 핵심 데이터 흐름

### 1. 채팅 메시지 처리 흐름

```mermaid
sequenceDiagram
    participant U as 사용자
    participant UI as ChatScreen
    participant VM as ChatViewModel
    participant PM as PromptBuilder
    participant AM as AiProviderManager
    participant NP as GeminiNanoProvider
    participant CP as GeminiCloudProvider
    participant DB as Room DB

    U->>UI: 메시지 입력
    UI->>VM: sendMessage(text)
    VM->>DB: 사용자 메시지 저장
    VM->>PM: buildPrompt(systemPrompt, history, userMsg)
    PM-->>VM: 조합된 프롬프트
    VM->>AM: generateStream(prompt)
    
    alt Gemini Nano 지원 기기
        AM->>NP: generateStream(prompt)
        NP-->>VM: Observable<String> (스트리밍 토큰)
    else 미지원 기기 → 클라우드 폴백
        AM->>CP: generateStream(prompt)
        CP-->>VM: Observable<String> (스트리밍 토큰)
    end
    
    VM-->>UI: 실시간 스트리밍 표시
    VM->>DB: AI 응답 저장
```

### 2. 멀티채널 메시지 라우팅 흐름

```mermaid
sequenceDiagram
    participant TG as Telegram
    participant CS as ChannelService
    participant MR as MessageRouter
    participant SM as SessionManager
    participant AM as AiProviderManager
    participant TM as ToolManager
    participant DB as Room DB

    TG->>CS: 인바운드 메시지
    CS->>MR: route(channel, sender, message)
    MR->>SM: getOrCreateSession(channel, sender)
    SM->>DB: 세션 조회/생성
    MR->>AM: generateResponse(session, message)
    
    alt AI가 도구 호출 결정
        AM-->>TM: callTool(name, params)
        TM-->>AM: ToolResult
        AM-->>MR: 최종 응답 (도구 결과 포함)
    else 직접 응답
        AM-->>MR: 텍스트 응답
    end
    
    MR->>CS: sendResponse(channel, response)
    CS->>TG: 응답 전송
    MR->>DB: 메시지 저장
```

### 3. 음성 대화 흐름

```mermaid
sequenceDiagram
    participant U as 사용자
    participant WW as WakeWordService
    participant STT as SttManager
    participant VM as VoiceChatViewModel
    participant AM as AiProviderManager
    participant TTS as TtsManager

    U->>WW: "Hey Claw" (웨이크 워드)
    WW->>STT: 리스닝 시작
    U->>STT: 음성 입력
    STT-->>VM: 인식된 텍스트
    VM->>AM: generateResponse(text)
    AM-->>VM: AI 응답 텍스트
    VM->>TTS: speak(response)
    TTS-->>U: 음성 출력
```

## 🔐 보안 아키텍처

| 영역 | 기술 | 설명 |
|---|---|---|
| **API 키 저장** | EncryptedSharedPreferences + Android Keystore | AES256-GCM 암호화 |
| **앱 잠금** | PIN + BiometricPrompt | 생체인증 우선, PIN 폴백 |
| **채널 인증** | DM 페어링 코드 | OpenClaw 패턴: 미인증 사용자 → 코드 발급 → 승인 |
| **Prompt Injection 방어** | 입력 산화 (sanitization) | 인바운드 메시지에서 시스템 프롬프트 오버라이드 시도 감지·차단 |
| **데이터 전송** | TLS 1.3 | 모든 클라우드 API/채널 통신 암호화 |
| **로컬 데이터** | Room (평문) + 선택적 SQLCipher | 기기 암호화(FBE) 기반, 추가 암호화 선택 |
